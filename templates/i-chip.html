Introduction

Sponsored by:
A VLSI event: Now there is no need to manually place gates and transistors to build electronic circuits. Describe complex circuits at an abstract level in terms of functionality and data flow by designing those circuits in HDLs. This FPGA Design Contest requires one to write the code for the given problem statement in either VHDL or Verilog and optimize it, then dump the code on the FPGA Kit to physically realize the design that was coded.

Problem Statement:

Participants have to detect edges in any given image.
An image will be provided in the form of a text file and the participants have to output a text file which represents an image with only the edges left from the original image.
Any HDL(VHDL/Verilog) can be used.
RTL design must be synthesizable and should be targeted for FPGA technology virtex7.
Pipelined designs are recommended.

Judging Criteria:

The design must cover all aspects of problem statement and shows correct simulation for all test cases.
Winning criterion is based on QofR (quality of result both in context of accuracy and run time performance).
